Introduction

MaaXBoard 8ULP features the NXP i.MX 8ULP processor to achieve ultra-low power, EdgeLock® secured intelligent edge applications.

The i.MX 8ULP device is architected with 3 separate processing domains: The application domain includes two Arm® Cortex®-A35 (1 GHz) cores plus 3D/2D GPUs for GUI-enabled Linux applications. The Real Time domain includes an Arm Cortex-M33 (216 MHz) core, plus Fusion DSP (200 MHz) core for low-power audio/voice use cases.

i.MX 8ULP SOM

  • Compact SOM dimensions (43mm x 36 mm)
  • 2x Samtec ST4 (0.4 mm pitch) 100pin board-to-board connectors
  • NXP i.MX 8ULP Processor
    • 2x Arm Cortex A35 (1.0 GHz)
    • 1x Arm Cortex M33 (216 MHz)
    • HiFi 4 DSP (600 MHz)
    • Fusion DSP (200 MHz)
    • 3D/2D-GPU (317 MHz)
    • 768 KB shared RAM (on-chip)
  • 2 GB LPDDR4X (32-bit)
  • 32 GB eMMC memory
  • 8 MB Octal SPI PSRAM
  • 4 MB Octal SPI NOR Flash
  • NXP PCA9460B PMIC

MaaXBoard 8ULP Carrier Board

  • Raspberry Pi form-factor
  • 2x Samtec SS4 connectors for 8ULP SOM
  • M.2 socket for Wi-Fi/BT module accessory
  • 1G Ethernet port (RJ45)
  • 2x USB 2.0 host, 1x USB 2.0 device ports
  • MIPI LCD interface
  • MIPI-CSI camera interface
  • Audio codec, DMIC and stereo jack I/O
  • 1x User RGB LED, 3x button switches
  • 40-pin HAT and 16-pin Click headers
  • 6-pin WTB Header (ADC in, DAC out)

Wi-Fi / BT Module

  • M.2 Key E socket
  • Accepts AzureWave or Panasonic M.2 modules

  • Automotive
    • Heating Ventilation, and Air Conditioning (HVAC)
  • Industrial
    • Building Safety, Building Security, Heat Metering, Industrial HMI, Smart Power
  • Mobile
    • Smart Watch, Smart City
  • Fleet Management
    • Inventory and Supply Chain Management, Transport Ticketing
  • Smart Home
    • Home Security and Surveillance, In-Home Energy Display, Major Home Appliances, Robotic Appliance, Small and Medium Appliances
